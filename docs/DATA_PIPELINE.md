# Data Build Pipeline

PriceOfElectricity.com uses a manual-input data pipeline with versioned snapshots. This document describes the available commands, how snapshots work, and how to update raw data.

## Commands

All commands are defined in `package.json` and run via `tsx`.

### `npm run data:validate`

Loads every state from `src/data/raw/states.raw.ts` and runs it through the validation layer (`validateRawState`). Checks that each state has a valid name, a positive numeric rate, and a parseable date.

- Prints a summary of passed/failed states.
- Exit code `0` if all states pass; `1` if any fail.

### `npm run data:build`

Validates and transforms all raw states, then writes a versioned snapshot:

- File: `src/data/snapshots/v<YYYYMMDD>.json`
- Also writes: `src/data/snapshots/latest.json` (identical content)

The snapshot contains:

```json
{
  "version": "v20260222",
  "releasedAt": "2026-02-22",
  "states": [
    { "slug": "alabama", "rate": 16.55, "updated": "February 2026" }
  ]
}
```

### `npm run data:diff -- <old> <new>`

Compares two snapshot versions and prints the top 10 largest rate increases and decreases.

```
npm run data:diff -- v1 v2
```

If run without arguments, lists available snapshot versions.

## How snapshots work

Snapshots are JSON files stored in `src/data/snapshots/`. Each file captures state-level average residential electricity rates at a specific point in time.

- `v1.json`, `v2.json` â€” manually created historical baselines
- `v<YYYYMMDD>.json` â€” generated by `data:build`
- `latest.json` â€” always points to the most recent build output

The `snapshotLoader.ts` library (`src/lib/snapshotLoader.ts`) loads snapshots at build time for the `/data-history` pages. When you add a new snapshot and want it visible on the site, register its import in `snapshotLoader.ts`.

## How to update raw data

1. Edit `src/data/raw/states.raw.ts` with new rate values and update dates.
2. Run `npm run data:validate` to confirm all entries are valid.
3. Run `npm run data:build` to generate a new versioned snapshot.
4. Register the new snapshot in `src/lib/snapshotLoader.ts` if you want it visible on `/data-history`.
5. Run `npm run build` to regenerate the static site.
